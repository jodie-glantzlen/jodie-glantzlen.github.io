Some Javascript concepts explained

HTML vs CSS: an analogy

Let’s say you have a completely empty house: no rooms, no floors, nothing. Think of HTML as a builder and CSS as an artist/painter. HTML will build your house a structure, by giving it different floors, rooms, furniture, etc. Then CSS will make your house colourful, organised and pretty! It will paint the walls, organise the layout of each room, make your furniture smaller or bigger, etc. Easy!

JS control flow and loops

Control flow in JavaScript is how your computer runs code from top to bottom. It starts from the first line and ends at the last line, unless it hits any statement that changes the direction.

An example of such statements would be loops. They're iteration statements that will keep running until there is either nothing left to loop over, or if the condition under which they run becomes false.

Don't understand a word I'm saying? Yeah, me neither. Let's start again with an example from our daily life. I'm French, and I love crêpes. So today, I'm gonna keep making crêpes until I run out of milk.

  • I'm making my first batch of crêpes.
  • Still got milk? Yup!
  • Let's make another one.
  • I'm making my second batch of crêpes.
  • Still got milk? Oui!
  • Okay, third batch incoming.
  • I'm making my third batch of crêpes.
  • Oh no. No milk left.
  • No more crêpes.

Tech explanation: Loops are conditional, meaning that it runs based on if a certain condition is true. As long as it stays true, the code is going to keep executing.

Non-tech translation: As long as there’s milk in my fridge, the crêpes are going to keep being made.
