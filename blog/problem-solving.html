Stuck on a simple problem

I have a rather strong personal vendetta against maths, so whenever I face a problem involving it, it gets complicated. When I was working on the Loops section of Javascript Intro, one of the exercises was meant to be solved with the *deep breaths* modulus operator. It was a simple problem: looping over an array and return a new one that includes every second element. Needless to say I was very grumpy, and, being properly French, I led a one-person protest from my desk and came up with my own, maths-free solution. Alas. The system wanted the modulus operator, and I couldn't pass the test without it.

I Googled my way out of that one. And I guess I'm glad I did, because it forced me to read about it and understand how it works. I would never have taken the time to research it had it not been for that exercise. Did I make peace with it though? Never.

Problem-solving techniques

Improving code with reflection and trying something

These two are interdependent in my opinion. At this stage of my coding journey there's only so much improvement I can do but I will implement it regularly once I progress further. What I like to do is trying something, test it, comment it, try something else, test it, and so on.

Googling

I use Google all the time. If the documentation doesn't help or if I don't understand it completely, I know there's probably a thousand people out there who faced a similar problem or tried to achieve what I want to achieve. Looking at other people's code is great for learning as well.

Rubber Ducky

As cute as the duck is, we haven't had the chance to solve a problem together yet.

Pseudocode

It doesn't really come naturally to me to write it down, it's mostly happening in my head at this stage. But I see how it can be helpful when used on bigger, more complex projects.

Reading error messages

Yup, these are 100% helpful. Obviously my first reaction when seeing them is "Oh no, what is it this time", but on several occasions they helped me figure out where my code was wrong and most importantly, what was wrong with it. Sometimes it's just a typo, but if you're not looking for it, good luck figuring out what's going on.

Console.logging

My best friend #2, with Google. It's like a translator that helps me communicate with my code. If my code is working, I can see it with console.log. If it's not, I can see it thanks to console.log too.

Asking peers/coaches for help

I try to ask for help only as a last resort, when I'm truly, fully stuck, when none of the above works, or when I run out of brain power.
